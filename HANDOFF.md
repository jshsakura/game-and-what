## What shipped this session

### Activity feed + trash recovery (the big one)
- **Append-only** event log: `events` table (`db.py`), `services/events.py` (`log`, `seed_uploads`, `recent`, `get`, `mark_restored`), `routers/events.py` (`GET /sessions/{sid}/events`, `POST .../events/{id}/restore`).
- Logged from: upload (`roms.py`), rename/delete (`manage.py`), pico8-compat/sd-exclude (`package.py`), 한글패치 (`lang.py`). All write inside the mutation's own transaction.
- **Restore**: delete stores the full row snapshot in `meta`; files go to `_trash` (`storage.move_to_trash`) and can be undone for **30 days** (`RETENTION_DAYS`) via `storage.restore_from_trash`. Startup runs `seed_uploads` (backfills history) + `purge_trash` (drops expired) + `packaging.prune_cache`.
- Frontend: `frontend/src/ActivityFeed.jsx` — bell button (far right of top bar, `icon-btn` style + subtle terracotta unread badge) → right drawer, newest-first, skeleton/empty states, restore button on delete rows. Korean i18n in `ko.js` only (other locales fall back → issue #4). Panel shows recent 50; table keeps everything (pagination = issue #3).
